# QO-1: 리포트 상태 플래그 및 롤백/숨김 계획

## 목표
- 리포트 상태(대기/게시/숨김)를 관리하고 잘못된 리포트를 한 번 클릭으로 숨기거나 롤백할 수 있게 한다.

## 범위
- 포함: 상태 필드/플래그 정의, 숨김/복구 액션 API+UI, 목록/상세에서 상태 반영.
- 제외: Slack/이메일 알림, 역할 기반 승인 흐름(후속 이슈).

## 요구사항
1) 상태 모델: draft(대기), published(게시), hidden(숨김). 기본 published.
2) 롤백/숨김: 운영자가 버튼으로 hidden 전환, 필요 시 draft 복구.
3) 표시: 목록/상세에서 상태 배지, hidden은 기본 조회 제외. “검토 중/숨김” 안내 문구.
4) 접근: 임시로 인증 없이 노출 가능하나, 추후 관리자만 허용할 수 있도록 플래그/미들웨어 훅을 남긴다.

## 태스크
- [ ] DB/모델에 상태 필드 추가(기본 published).
- [ ] API: 상태 업데이트 엔드포인트 또는 기존 favorite API와 별도 패스 추가.
- [ ] UI: 목록/상세에 배지, 숨김/복구 버튼, hidden 시 비노출(필터 옵션으로 포함 여부 선택).
- [ ] 테스트: 상태 변경 후 목록/상세 반영, 숨김 시 기본 조회 제외, 복구 시 재노출.

## 리스크
- 인증 부재로 누구나 상태를 바꿀 수 있음 → 임시 보호 토큰/플래그 고려.
- 캐시/검색 인덱스가 있다면 동기화 지연 가능 → 현재는 DB 조회 기준으로 단순 처리.
